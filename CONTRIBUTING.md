# Contributing to MIDITypist

First off, thank you for considering contributing to MIDITypist! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue using the **Bug Report** template. Provide as much detail as possible, including:
- Steps to reproduce
- Expected vs Actual behavior
- Your OS and MIDI device information

### Suggesting Enhancements
We love new ideas! Use the **Feature Request** template to suggest improvements.

### Pull Requests
1.  Fork the repository.
2.  Create a branch for your feature (`git checkout -b feature/AmazingFeature`).
3.  Commit your changes (`git commit -m 'Add some AmazingFeature'`).
4.  Push to the branch (`git push origin feature/AmazingFeature`).
5.  Open a Pull Request.

## Development Setup

1.  Clone the repository: `git clone https://github.com/SamuelJoseph23/MIDITypist.git`
2.  Open `main/MIDI Mapper.sln` in **Visual Studio 2022**.
3.  Install necessary NuGet packages:
    - `Microsoft.Web.WebView2`
    - `Microsoft.Windows.ImplementationLibrary`
4.  Build the solution in **Debug** or **Release** (x64).

## Style Guide
- Follow existing C++ and JavaScript naming conventions.
- Keep the UI glassmorphic aesthetics consistent.
- Document new features in the README or relevant files.

Thank you for contributing!
